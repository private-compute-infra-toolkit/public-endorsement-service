# Public Endorsement Service (PES)

The **Public Endorsement Service (PES)** is a Java service designed to provide verifiable endorsements for artifacts.
It allows publishers to submit signed statements (such as [in-toto](https://in-toto.io/) statements) and receive a
PES-signed endorsement that is recorded in a Transparent Ledger. The endorsement provides verifiable proof that the
statement is publicly accessible.

## Overview

PES acts as a trusted third-party witness in the software supply chain. When a publisher creates an artifact and a
corresponding statement (e.g., a build provenance), they can submit it to PES. PES then:

1. **Validates** the publisher's identity and signature.
2. **Records** the statement in a **Transparent Ledger (TLedger)** to ensure immutability.
3. **Endorses** the combined statement, publisher signature, and TLedger receipt by signing them with the PES service
   key.

This process provides strong guarantees about when a statement was created and that it hasn't been tampered with since
its publication.

## Root of Trust

PES is built on top of the Measurement Bound Storage (MBS) library. The root of trust for PES is the
measurement-bound certificate generated by MBS. This certificate binds the PES service's identity to the specific code
measurement of the Trusted Execution Environment (TEE) in which it runs, ensuring that only authorized versions of PES
can perform signing operations.

## Key Features

- **Verifiable Endorsements**: Generates cryptographically signed endorsements using Pre-Authentication Encoding (PAE)
  to ensure integrity.
- **Transparent Ledger Integration**: Automatically logs every endorsement to a transparent ledger, providing
  a public, tamper-evident audit trail.
- **In-toto Support**: Natively
  supports [in-toto v1 statements](https://github.com/in-toto/attestation/tree/main/spec/v1) for describing software
  supply chain metadata.
